기초자료 정리 및 요약

1. 기초자료 정리

  • 제공된 정보에서 신청서 관련 정보만 추출
  • 불필요한 정보 제거 및 관련 정보 선별
  • 분야별 정보 분류 작업
  • AI가 처리할 수 있는 형태로 데이터 가공

2. 기초자료 요약

  • Google NotebookLM 활용하여 요약 작업
  • 분야별 요약 정보 작성
  • "학습자료(참고자료 요약자료).zip" 생성
  • 방대한 원본 자료를 AI가 효율적으로 처리할 수 있는 분량으로 축소

양질의 기초자료 선별과 요약이 AI 작업 효율성의 핵심

기초자료 요약 누락 문제

사실 확인 문제
외부 우수사례 제공으로 인한 비사실적 정보 포함 → 사실 확인 필요성 대두

현 보고서의 한계

식별된 주요 한계점

  • 구체성 부족 - 요약자료 기반 작성으로 인한 세부 내용 부족
    • 보고서 작성자의 추가 보완 필요
  • 자료 부족 영역 - 평가 기준에 맞는 추가 자료 필요
    • 제목만 적시된 부분 (예: 급여 관련)
    • 작성자가 추가 자료 확보 필요
  • AI 착란현상(Hallucination) - 비사실적 내용 생성 가능성
    • 사실 확인 과정 필수
    • 실제 데이터와의 대조 검증 필요

AI는 보조 도구로서 활용하고, 최종 검증은 담당자가 수행해야 함

총평 및 시사점

가능성

  • 평가지표에 맞춘 스토리텔링 중심 보고서 작성 가능
  • 사전 자료 선별 시 생산성과 효율성 향상
  • 가이드라인 활용으로 보고서 틀 제공 가능

한계점

  • 방대한 데이터 처리에 따른 토큰 소모 증가
  • AI 착란현상(Hallucination) 발생 위험
  • 세부적인 구체성 부족 문제

    • 시사점: AI를 활용한 보고서 작성은 단순 작업이 아닌 복합적 프로세스이며, AI는 인간 작업자를 대체하는 것이 아닌 보조하는 도구로 활용할 때 가장 효과적

    AI 활용 업무 시 권장사항

    현업 적용을 위한 베스트 프랙티스

    1. 철저한 사전 준비 - 관련 자료 선별 및 요약 작업 필수
    2. 명확한 기준 설정 - 평가 지표와 작성 방향 구체화
    3. 단계적 접근 - 전체 작업을 단계별로 나누어 진행
    4. 지속적 사실 확인 - AI 생성 내용의 사실 여부 검증
    5. 인간-AI 협업 모델 - AI는 초안 작성, 인간은 검증 및 보완
    6. 반복적 개선 - 피드백을 통한 지속적인 품질 향상

    AI는 보조 도구이며, 최종 책임은 항상 사람에게 있다
